    命令执行的方式

    • command [-options] parameter1 parameter2 …
      • []实际并不存在，-加入为参数，如-h;–为完整参数名,如–help;使用\转义回车键，适用于命令过长。
    • bc:计算器命令
    • tab键：连按两下补齐命令、文件
    • 在线帮助：man page和info page
      • 1代表一般账号可用命令,8代表系统管理员常用命令,5代表系统配置文件格式
    • 关机命令
      shutdown:-c 取消shutdown
      reboot:等同于shutdown -r now
    • 要取得终端支持的语言可执行echo $LANG或locale

    文件权限与目录配置

    1. ls -al：列出所有文件包括隐藏文件。

    2. -rwxrwx—

    • 第一个字符
      • d:目录
      • -:文件
      • l:链接文件
      • b:设备文件里的可供存储的接口设备
      • c:设备文件里的串行端口设备
    • 第一组：文件所有者权限
    • 第二组：同用户组的权限
    • 第三组：其他非本用户组的权限
    • 修改属性与权限
      • chgrp:改变文件所属用户组
      • chown:改变文件所有者
      • chmod:改变文件的权限

    3. 目录属性

    • r:具有读取目录结构列表的权限
    • w:具有更改该目录结构列表的权限
    • x：用户能否进入该目录成为工作目录

    4. 目录配置

    • /etc:开机与系统数据文件均在这个目录之下
    • /bin,/sbin,/usr/bin,/usr/sbin:这是系统预设的执行文件的放置目录
    • /usr/local:这是系统预设的让你安装你后来升级的套件的目录
    • /home：这个是系统将有账号的人的家目录设置的地方
    • /var:www,ftp,db,mail,logmessages…
    • /usr/share/man,/usr/local/man:这两个目录为放置各类套件说明档的地方

    Linux文件和目录管理

    1. 目录与路径

    • 目录

      • .代表当前所在目录
      • ..代表上层目录
      • -表示前一个工作目录
      • ~代表home的目录
      • ~user代表user这个人的家目录

    • 命令

      • cd 变换目录
      • pwd 当前目录
      • mkdir 建立目录：-m,配置文件案的权限；-p,递归创建。
      • rwdir 删除目录：仅能删除空目录；-p,递归删除。
      • 环境变量PATH：当执行一个指令时，系统会依照PATH定义的路径搜寻档案

    2. 文件与目录管理

    • 查看文件与目录：ls

    • 复制、删除、移动：cp、rm、mv

      • cp:-a,相当于pdr;-p,使用文件属性非默认属性复制；-d,若为链接文件,复制链接文件属性非文件本身;-r,递归复制，用于目录。
      • rm:-f,强制删除；-i,询问；-r,递归删除。
      • mv:-f,强制移动不询问；-u,若目标文件已经存在且源文件比较新，才会更新；-i,若目标文件已经存在，就会询问是否覆盖。
      • basename,dirname

    3. 查看档案内容

    • cat:由第一行开始显示档案内容
      • -n:显示行号
      • -b:显示行号（仅非空白行）
    • tac:由最后一行开始显示档案内容
    • nl:显示时输出行号
    • more：一页一页显示
    • less:类似more且可翻页
    • head、tail:只看头、尾几行
    • od:二进制显示
    • touch:修改文件时间与创建新文件
      • -a:修改访问时间atime
      • -c:修改文件的时间状态时间
      • -m:修改mtime修改时间

    4. 文件与目录的默认权限与隐藏权限

    • 查看：umask,umask -S。
    • 改变文件隐藏属性:chattr [+-=][ASacdistu] 文件或目录
      • +-=：增加、减少、设定属性
      • a:只能增加数据，不能删除修改数据，只有root才能这只这个属性
      • i:不能删除、改变、设定链接也无法写入或新增数据，只有root才能这只这个属性
      • 显示文件隐藏属性：lsattr

    5. 文件特殊权限

    • SUID:4、SGID:2
      • SUID:会创建s权限，让一般用户在执行某些程序时，暂时具有该程序拥有者的权限，他是程序在执行过程中拥有文件拥有者的权限，因此，他仅用于二进制文件；对目录无效。
      • Set GID:s的权限在用户组，就是Set GID。如果SGID用在二进制文件上，则不论用户是谁，在执行程序时，他的有效用户组将会变成该程序的用户组；如果SGID用在目录A上，则在A目录内所建的文件或目录的用户组，则会是次A目录的用户
    • Sticky bit:1
      • 只针对目录有效。SBit对目录的作用是：在具有SBit的目录下，用户若在该目录下具有wx权限，则当用户在该目录下建立文件或目录时，只有文件拥有者与root才有权利删除。
    • 设置大小写区别：大写表空
    • file:查看文件类型

    6.搜寻文件或目录

    • which :查看可执行文件的位置
      • -a:将所有可找到的命令列出
    • whereis:查看文件的位置
      • -b:只找binary的文档
      • -m:只找在说明文件manual路径下的文件
      • -s:只找source来源文件
      • -u：没有说明档的文件
    • locate: 配合数据库查看文件的位置
      • locate：依据/var/lib/mlocate内的数据库记载，找出用户输入的关键字文件名
      • updatedb: 根据/etc/updatedb.conf的设置去查找系统硬盘内的文件名，并更新/var/lib/mlocate内的数据库文件。
    • find:实际搜寻硬盘去查询文件名称

    Linux磁盘与文件系统管理

    1. EXT2文件系统

    • 目录：在Linux ext2文件系统建立一个目录时，ext2文件系统给该目录一个inode和至少一个block。inode记录目录属性，块记录这个目录下文件或目录的相关性
    • 文件：ext2至少给文件分配一个inode和相对于该文件大小的块的数量
    • inode：本身不记录文件名，仅记录文件的相关属性，文件名则记录在目录所属的块区域
    • 使用dumpe2fs查看inode等的信息
    • 载入点：必须是一个目录，不能是一个文件
    • Linux支持的文件系统查看：/proc/filesystems或/lib/modules/‘uname -r’/kernel/fs
    • Linux文件系统运行：异步方式

    2. 文件系统的简单操作

    • df [-ahikHTm][目录或文件名]：显示系统当前磁盘的总容量、已使用容量和剩余容量
      • -a:列出所有文件系统
      • -h:易读显示
      • -i:以inode数量显示
    • du [-ahskm] 档案或目录名称：查看磁盘、目录的大小
      • -s:列出总量，而不列出每个目录占用的容量
    • ln [-sf] 来源文件 目标文件
      • -s:不加参数为hard link，-s为symbolic link
      • -f:覆盖目标文件
      • hard link:不能跨文件系统，不能连接到目录

    3. 磁盘的分割、格式化、检验与挂载

    • fdisk [-l] 装置名称
      • -l:输出后面接的设备所有的分区内容。若仅有fdisk -l时，则系统将会把整个系统内能够搜寻到的装置的分区全部列出来。
    • 磁盘格式化
      • mkfs [-t 文件系统格式] 设备文件名
      • mke2fs [-b block大小] [-i block大小] [-L卷标] [-cj] 设备
    • 磁盘检验
      • fsck [-AtCary] 装置名称：系统出现极大的问题，导致你在Linux开机的时候进入单人单机模式下进行维护的行为时，才使用fsck；被检查的partition务必不可挂载到系统上，即在卸载的状态下进行检查
      • badblocks -[svw] 装置名称
      • sync:将系统暂存在内存中的数据回存写入磁盘当中
    • 磁盘挂载与卸载
      • mount [-tonL] 装置名称代号 挂载点
        • -a:依照/etc/fstab的内容将所有相关的磁盘都挂上来

    文件的压缩与打包

    1. Linux常见的压缩文件的后缀

    • *.Z compress程序压缩的文件
    • *.bz2 bzip2程序压缩的文件
    • *.gz gzip程序压缩的文件
    • *.tar tar程序打包的数据，没有压缩
    • *.tar.gz或tgz tar程序打包并经过gzip压缩的文件
    • *.tar.bz2 tar程序打包并经过bzip2压缩的文件
    • *.zip 使用zip压缩
    • *.rar 使用rar压缩

    2. compresse [-dcr] 文件或目录

    • -d:解压缩
    • -r:连同目录下的文件同时压缩
    • -c:将压缩数据输出为标准输出（输出到屏幕）

    3. gzip,zcat

    • -c，-d，-t：可以用来检验一个压缩文件的一致性，看文件有无错误
    • -#：压缩等级，1最不好，9最好，6为默认。

    4. bzip2,bzcat

    • -c，-d，-#，-z:压缩的意思
    • bzcat file.bz2:读取file的内容
    • .bz,.bz2,.tb2,.tbz2都可以使用bzip2解压缩

    5. tar

    • -z:是否同时具有gzip的属性
    • -j:是否同时具有bzip2的属性
    • -x:揭开一个压缩文件的参数命令
    • -t:查看tarfile里面的文件
    • -c:创建一个解压缩文件的命令
    • -v:压缩的过程中显示文件
    • -f:使用文件名，在f后立即接文件名
    • -p:使用原文件的原来属性（属性不会依据以后改变）
    • -P:可以使用绝对路径
    • –exclude FILE：在压缩的过程中，不要将FILE打包

    6. zip

    • zip [AdFmrSuz#] 目标文件名 源文件名
    • -A：调整可执行的自动解压缩文件
    • -d：从压缩文件中删除指定的文件
    • -F：尝试修复已损坏的压缩文件
    • -m：将文件压缩并加入压缩文件后，删除原始文件，即把文件移动到压缩文件中
    • -r：递归处理
    • -S：包含系统和隐藏文件
    • -u：更换较新的文件到压缩文件内
    • -z：替压缩文件加上注释
    • -#

    7. zipinfo

    • zipinfo [-lhMtz] [压缩文件][文件…]
    • -l:只列出文件名
    • -h:只列出压缩文件的名称
    • -M:若信息内容超过一个画面，则采用类似more指令的方式列出来
    • -t:只列出压缩文件内所包含的文件数目，压缩前后的文件大小及压缩率
    • -z:如果压缩文件内有注释，就将注释显示出来

    8. unzip

    • unzip [-ltvMP] [.zip][文件][-d目录]
    • -l:显示压缩文件内所包含的文件
    • -t:检查文件是否正确
    • -v:显示执行时的详细信息
    • -M:将输出的结果送到more程序处理
    • -P<密码>：使用zip密码选项
