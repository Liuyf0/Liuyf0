Java的初步学习之继承
2020-02-12

1.继承

  • 1.利用继承，可以基于一个已知的类构造一个新类。使用关键字extends表示继承，已存在的类为超类，新类为子类。子类比超类拥有的功能更加丰富。

  • 2.在通过扩展超类定义子类时，仅需要指出子类与超类的不同之处。因此在设计类的时候，应该将通用的方法放在超类中，将具有特殊用途的方法放在子类中。

  • 3.super：

    • 1）.使用super调用构造器的语句必须是子类构造器的第一条语句。
    • 2）.子类调用超类中的方法。

  • 4.置换法则：程序中出现超类对象的任何地方都可以用子类对象置换。如,可以将一个子类的对象赋给超类变量。

    在这个例子中，可以这样调用：boss.setBonus(5000); // OK
    但不能这样：staff[0].setBonus(5000); // ERROR

  • 5.阻止继承：有时候，可能希望阻止利用某个类定义类，不允许扩展的类称为final类。如果在定义类时使用了final修饰符就表明这个类是final类。类中的方法也可以被声明为final，此时，子类就不能覆盖这个方法（final类中所有的方法自动成为final方法）。

  • 6.只能在继承层次内进行类型转换。

  • 7.在将超类转换成子类之前，应该使用instanceof进行检查。

2.多态

一个对象变量可以引用多种实际类型的现象。在运行时能够自动地选择调用哪个方法的现象称为动态绑定。

  • 1.如果在子类中定义了一个与超类签名相同的方法，那么子类中的方法就覆盖了超类中的相同签名的方法。
    • 签名：方法的名字和参数列表称为方法的签名。
  • 2.对于private、static、final或者构造器，编译器准确知道应该调用哪个方法，称为静态绑定。

3.抽象类

  • 1.使用abstract关键字，如：public abstract String getDescription();。包含一个或多个抽象方法的类必须被声明为抽象类。
  • 2.类即使不含抽象方法，也可以将类声明为抽象类。
  • 3.抽象类不能被实例化。也就是说，如果将一个类声明为abstract，就不能创建这个类的对象，但可以创建具体子类的对象。需要注意，可以定义一个抽象类的对象变量，但是它只能引用非抽象子类的对象。如：new Person("Vince Vu") // error, Person p = new Student("Vince Vu","Economics") // ok

    5.泛型数组列表

  • 1.ArrayList类

    • 1).一个采用类型参数的泛型类。为了指定数组列表保存的元素对象类型。需要用一对尖括号括起来加在后面，如：ArrayList<Employee>。声明和构造一个保存Employee对象的数组列表：ArrayList<Employee> staff = new ArrayList<Employee>();
    • 2).使用add,remove方法可以将元素添加到数组列表或删除一个元素，如：staff add(new Emplyee(...));，Employee e = staff.remove(n);,指定位置添加元素，后面元素后移：staff.add(n, e);
    • 3).能够估计元素数量，如：staff.ensureCapacity(100);或ArrayList<Employee> staff = new ArrayList<Employee>(100);
    • 4).返回数组列表中包含的实际元素数量：staff.size()
    • 5).trimToSize,确认数组大小，回收多余存储空间。
    • 6).使用get和set方法实现访问或改变数组元素的操作，如：staff.set(i, harry);,Employee e = staff.get(i);。
    • 7).toArray:拷贝数组，如：list.toArray(a);

6.对象包装器与自动打包

  • 1.包装器：Integer、Long、Float、Double、Short、Byte、Character、Void、Boolean。对象包装器是不可变的，即一旦构造了包装器，就不允许更改包装在其中的值。同时，包装器类还是final，因此不能定义其子类。
  • 2.定义整形数组列表：ArrayList<Integer> list = new ArrayList<Integer>();
  • 3.自动打包：list.add(3);自动转换为list.add(new Integer(3));
